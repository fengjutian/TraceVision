# 鸿蒙应用开发宪法（HarmonyOS App Constitution）

## 技能目的

为鸿蒙 HarmonyOS 应用开发团队提供统一的技术原则、代码规范、安全要求与质量标准。所有代码生成、审查、重构任务均须以本宪法为准绳，确保应用的可维护性、安全性与一致性。

## 触发场景

- 用户请求生成鸿蒙应用代码（ArkTS / ArkUI / FA / Stage 模型）
- 用户请求代码审查或重构建议
- 用户询问鸿蒙架构设计、目录结构、组件拆分方案
- 用户询问权限申请、数据安全、网络安全相关规范
- 用户询问测试策略、性能指标或 CI/CD 质量门禁

## 参考文档

加载详细规范前，先读取以下参考文件：

- `references/harmonyos-principles.md` — 完整的技术原则、代码规范、安全要求与质量标准

## 工作流程

### 代码生成任务

1. 读取 `references/harmonyos-principles.md` 中对应章节
2. 按【技术原则】确定架构模式与模块划分
3. 按【代码规范】生成符合命名约定、注释要求的 ArkTS/ArkUI 代码
4. 按【安全要求】检查权限声明、数据处理与网络调用
5. 按【质量标准】确认代码复杂度、测试覆盖率要求已满足

### 代码审查任务

1. 读取 `references/harmonyos-principles.md` 全文
2. 逐项核对代码规范符合度
3. 输出审查报告，格式如下：
   - **[违规]** 具体问题 → 修正建议
   - **[警告]** 潜在风险 → 优化建议
   - **[通过]** 符合规范的亮点

### 架构设计任务

1. 优先参考【技术原则 § 架构设计原则】
2. 输出分层架构图（文字描述）与目录结构建议
3. 说明各层职责与通信规约

## 关键约束（必须遵守）

- 禁止使用 `any` 类型；所有变量必须显式声明类型
- 禁止在 UI 层直接进行网络请求或数据库操作
- 权限最小化原则：只申请功能所需的最低权限
- 所有用户敏感数据必须加密存储，禁止明文写入文件或日志
- 单个函数不超过 50 行，圈复杂度不超过 10
